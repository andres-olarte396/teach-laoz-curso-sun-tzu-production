# PROYECTO FINAL: EL PLAN DE BATALLA ESTRATÉGICO

## OBJETIVO

Aplicar todo el conocimiento del curso ("Brainset Sun Tzu") para analizar, planificar y diseñar la estrategia de un sistema de software complejo del mundo real.

## ESCENARIO

Has sido contratado como **CTO interino** para salvar "FinTech-X", una startup de pagos que está a punto de quebrar por deuda técnica y caos organizativo.

### Situación Actual (Intelligence Report)

1. **Producto**: App de pagos móviles.
2. **Infraestructura**: Monolito en Java 7 hospedado en servidores físicos (On-premise).
3. **Equipo**: 20 desarrolladores quemados (Burnout). Alta rotación.
4. **Proceso**: Deploys manuales cada 3 semanas. Tasa de fallo del 40%.
5. **Negocio**: El CEO quiere lanzar "Crypto-trading" en 2 meses para levantar una ronda de inversión.
6. **Incidentes**: Caídas diarias a las 18:00 sin causa conocida. No hay logs centralizados.

---

## TU MISIÓN (ENTREGABLES)

Debes redactar un documento (`PLAN_BATALLA.md`) con las siguientes 4 secciones:

### 1. Análisis de los 5 Factores (Módulo 1)

- **Tao**: ¿Está el equipo alineado con la misión Crypto del CEO? Diagnostica.
- **Clima**: ¿Es el momento de lanzar Crypto con el sistema actual?
- **Terreno**: Clasifica el terreno técnico (¿Mortal, Enredado?).
- **Liderazgo**: ¿Qué virtudes faltan actualmente?
- **Método**: Evalúa el proceso de deploy actual.

### 2. Estrategia Arquitectónica (Módulo 2)

- **Defensa**: Propón una estrategia para salir de los servidores físicos hacia Inmutabilidad (Cloud/Docker). ¿Big Bang o Strangler Pattern?
- **Orquestación**: Define qué partes serán Zheng (Aburridas) y cuáles Qi (Innovadoras) para la nueva feature Crypto.
- **Foco**: Identifica el "Vacío" donde atacar la deuda técnica primero.

### 3. Operaciones de Combate (Módulo 3)

- **Maniobra**: Diseña un MVP "Mago de Oz" o "Fake Door" para la feature Crypto que no requiera tocar el monolito Java.
- **Inteligencia**: Define las 3 primeras métricas (Golden Signals) que implementarás para diagnosticar las caídas de las 18:00.
- **Protocolo**: Establece la regla de "Error Budget" para frenar los lanzamientos si la estabilidad no mejora.

### 4. Ofensiva Política (Módulo 4)

- Escribe el guión de la conversación con el CEO para explicarle que "Crypto en 2 meses" en el monolito actual es imposible, pero ofreciéndole un "Puente Dorado" (Alternativa viable).
